# Instagram Carousel

## КРИТИЧНО: автостарт при активации

Когда скилл срабатывает — сразу переходи к шагу 1, без пересказа этого файла.

## Шаг 1. Сбор входных данных

Спроси:

> Вставь контент для карусели (пост, тезисы, конспект) или опиши тему, чтобы я написал текст с нуля. Если есть 2-3 карусели/слайда, дизайн которых нравится — пришли их, я вытащу из них дизайн-систему (палитру, шрифты, отступы) и буду использовать её. Если есть свои фото или видео (например, для карусели-отчёта о поездке) — пришли их, я подберу текст под каждое фото/клип и сделаю карусель с медиа на фоне.

Если пользователь дал папку с несколькими фото и порядок не очевиден (травел-карусель, репортаж) — предложи отсортировать их по времени съёмки:

### Голос письма (tone-of-voice)

`brand-kit.md` выше — про визуальный стиль. Отдельно от него проверь наличие голосового референса — источника, как именно писать текст:

1. `voice.md` в корне проекта;
2. если его нет — `tone-of-voice.md` в корне проекта;
3. если пользователь в диалоге явно называет источник голоса по имени (например, «голосом как в tone-of-voice», «как у Зои», ссылается на конкретный файл/скилл) — прочитай именно его, даже если он не лежит в корне проекта.

Если ничего не найдено и не названо — молча продолжай с нейтральным тоном, не спрашивай об этом отдельным вопросом (голос — не обязательная часть флоу, в отличие от формата/стиля/числа слайдов).

Если референс найден — прочитай его целиком и используй в Шаге 2 и Шаге 4, как описано там. Это не про палитру/шрифты (за это отвечает `brand-kit.md`/пресеты) — только про то, какими словами и с какой интонацией писать.

## Шаг 2. Дизайн-бриф

Построй слайд-бриф по структуре hook -> value -> CTA:

- **Слайд 1 (обложка)**: hook, крупный заголовок, без длинного текста
- **Слайды 2..N-1 (value)**: одна мысль на слайд, ≤25 слов текста
- **Слайд N (CTA)**: призыв подписаться/сохранить, визуально выделен

Для каждого слайда укажи номер, заголовок (≤8 слов), текст (≤25 слов), краткий alt-текст (описание содержимого слайда для доступности — не то же самое, что headline/body) и caption — видимую подпись под слайдом при пролистывании (см. «Per-slide captions» в Шаге 4). Три текстовых слоя разные: headline/body вшиты в картинку и жёстко ограничены по словам, altText невидим (только для скринридеров), caption виден при свайпе и не ограничен лимитом картинки — но всё ещё про содержание конкретно этого слайда, а не мини-версия общей подписи поста.

Если текст пишется с нуля (пользователь не вставлял свой контент) и найден голосовой референс (см. Шаг 1) — используй из него словарь (характерные слова/выражения, запрещённые слова и обороты), регистр обращения к читателю и узнаваемые речевые приёмы. Не пытайся втиснуть в заголовок/текст слайда структуру целого поста из референса (открывающий крючок, закрывающий вопрос, порядок абзацев, место хэштегов) — на ≤8/≤25 словах для этого физически нет места, и это не нужно: полная структура применяется в подписи (Шаг 4). Если пользователь вставил свой готовый контент — просто адаптируй его текст под лимиты слайдов, без переписывания под голос.

Скажи пользователю:

> Вот бриф по слайдам. Скажи, что поправить, или "генерируй", если всё устраивает.

Дождись явного подтверждения. Не переходи дальше без него.

## Шаг 3. Рендер

### Если выбран Gemini-режим

Как в `gemini-carousel`: один промпт на слайд в отдельном блоке кода, с одинаковыми brand-constraints (палитра, шрифт, стиль) во всех промптах для консистентности. В конце предложи вариант "один общий промпт на всю карусель".

### Если выбран автоматический рендер

Сначала определи среду выполнения:

```bash
node -v && npx --yes playwright --version
```

**Node и Playwright доступны (обычно Claude Code)** — путь `build_carousel.js`, одна команда на всю карусель, включая смешанные текст/фото/видео:

1. Собери манифест `slides.json`:
   ```json
   {
     "format": "portrait",
     "slides": [
       { "type": "text",  "headline": "...", "body": "...", "altText": "...", "caption": "..." },
       { "type": "photo", "source": "C:\\...\\photo.jpg", "headline": "...", "body": "...", "altText": "...", "caption": "..." },
       { "type": "video", "source": "C:\\...\\clip.mp4", "duration": 8, "headline": "...", "body": "...", "altText": "...", "caption": "..." }
     ]
   }
   ```
   `format`: `"portrait"` (1080x1350) или `"square"` (1080x1080). Для `video` поле `start` можно не указывать вовсе — тогда скрипт сам через ffprobe определит длительность исходника и возьмёт середину клипа вместо всегда 0-й секунды; `duration` без явного значения берётся из `design-system.json.video.defaultDurationSeconds` (8с) и в любом случае зажимается в 3-60с.
   Если пользователь дал свои hex-коды, дизайн-систему из референсов или выбрал пресет — примени их к копии `design-system.json` (или передай смерженный файл третьим аргументом на шаге 2), не трогая оригинал скилла.
2. Запусти:
   ```bash
   node <путь_до_скилла>/scripts/build_carousel.js slides.json ./carousel-output [design-system.json]
   ```
3. Скрипт сам разбирает тип каждого слайда, рендерит нужным способом (текст/фото через Playwright-скриншот, видео через ffprobe-автодетект + chroma-key оверлей + ffmpeg), кладёт `slide-01.(png|mp4) ... slide-N.(png|mp4)`, `carousel.zip` (PNG и MP4 вперемешку по номерам), `alt-text.json` (alt-текст на каждый слайд, из поля `altText` манифеста) и `slide-captions.json` (видимые подписи под слайдами, из поля `caption`; пустая строка, если слайд без caption) в `./carousel-output/`. При первом использовании видео может потребоваться разовая установка браузера: `npx playwright install chromium`.

Низкоуровневые скрипты `render.js` (только текст/фото), `render_video.js` (один видео-слайд) и `lib.js` (общие хелперы) остаются доступны напрямую — используй их вместо `build_carousel.js`, только если нужно перерендерить один конкретный слайд точечно, не пересобирая всю карусель.

**Node/Playwright недоступны (Claude.ai / Cowork, песочница без сети)** — путь `render_pillow.py`:

1. Собери `slides.json` — массив объектов `{"headline": "...", "body": "...", "altText": "...", "caption": "...", "format": "portrait"|"square"}` (format одинаковый для всех слайдов, portrait = 4:5, square = 1:1). Видео здесь не поддерживается (см. правило ниже).
2. Запусти:
   ```bash
   python <путь_до_скилла>/scripts/render_pillow.py slides.json <путь_до_скилла>/templates/design-system.json ./carousel-output
   ```
3. Скрипт нарисует те же слайды напрямую через Pillow (без HTML/браузера) по тем же константам дизайн-системы и создаст `slide-01.png ... slide-N.png` + `carousel.zip` + `slide-captions.json` (из поля `caption`, пустая строка если не задано).

Если ни `node`, ни `python` не запускаются — сообщи об этом пользователю и предложи Gemini-режим как запасной вариант.

### Видео-слайд — детали (работает только в Claude Code через `build_carousel.js`, нужен ffmpeg)

Видео-слайды не рендерятся с нуля — берётся уже существующий клип пользователя, обрезается и получает тот же текстовый оверлей, что и фото-слайды, через chroma-key (`templates/overlay-template.html` на сплошном фоне `design-system.json.video.chromaKey`, дальше `colorkey` в ffmpeg — CLI-скриншот Playwright не отдаёт альфа-канал напрямую). Не применимо в Claude.ai/Cowork (там нет доступа к локальным видеофайлам пользователя и, как правило, нет ffmpeg). `build_carousel.js` делает всё это сам по манифесту; ручной пошаговый вызов ниже нужен только для точечной пересборки одного видео-слайда:

1. Проверь длительность и разрешение исходника: `ffprobe -v error -show_entries stream=width,height,duration -of default=noprint_wrappers=1 <видео>` (внутри `build_carousel.js` это делает `lib.probeVideo`).
2. Уточни у пользователя (если не задано) начальную секунду отрезка и длительность — от 3 до 60 секунд (жёсткий лимит Instagram для видео-слайда карусели).
3. Заполни `templates/overlay-template.html` теми же плейсхолдерами, что и `slide-photo-template.html` (headline/body/dots/arrow), но без `{{PHOTO_URL}}` — вместо него `{{CHROMA_KEY}}` = `design-system.json.video.chromaKey` в формате `#RRGGBB` (например `0xFF00FF` → `#FF00FF`). Сохрани как `overlay-NN.html`.
4. Отрендери оверлей в PNG тем же способом, что и обычные слайды:
   ```bash
   npx playwright screenshot --viewport-size "<ширина>,<высота>" file:///<путь>/overlay-NN.html overlay-NN.png
   ```
5. Собери видео-слайд:
   ```bash
   node <путь_до_скилла>/scripts/render_video.js <исходное_видео> overlay-NN.png slide-NN.mp4 <ширина> <высота> <старт_сек> <длительность_сек>
   ```
   Скрипт обрежет клип по времени, смасштабирует+закадрирует под целевой размер (аналог `background-size: cover`), наложит оверлей через chroma-key (`colorkey` в ffmpeg) и закодирует в H.264/AAC с `+faststart`. Длительность автоматически зажимается в диапазон 3-60с, если передано другое значение.

При смешанной карусели (часть слайдов — фото/текст, часть — видео) финальные файлы называются по общей нумерации `slide-01.png`, `slide-02.mp4`, ... — порядок в имени файла определяет порядок в карусели. `build_carousel.js` упаковывает PNG и MP4 в один `carousel.zip` сам; при ручной пересборке через `render.js`/`render_video.js` по отдельности собери zip последним шагом самостоятельно.

## Шаг 4. Подпись и хэштеги

Отдельно от слайдов сгенерируй текст подписи к посту (короткий крючок + расширение мысли + CTA в конце) и 5-10 релевантных хэштегов. Это выдаётся дополнительно к слайдам — не встраивается в изображения. `alt-text.json`, который создаёт `build_carousel.js`/`render_pillow.py`, — отдельная вещь: alt-описание для каждого слайда (доступность), не подпись к посту.

### Per-slide captions (Instagram Multiple Captions)

С июня 2026 Instagram позволяет при публикации карусели переключить режим подписи: **одна подпись на весь пост** (как выше) ИЛИ **своя подпись на каждый слайд** (до 20 слайдов, показывается снизу экрана при пролистывании). Это выбор пользователя в интерфейсе Instagram на публикации, не в скилле — поэтому готовь **оба варианта всегда**: обычную единую подпись (как описано выше) и `caption` на каждом слайде брифа (Шаг 2), который собирается автоматически в `slide-captions.json`. Per-slide caption — не мини-версия общей подписи и не дубликат headline/body с картинки: это отдельный текст про содержание конкретно этого слайда, без ограничения в 25 слов.

Если найден голосовой референс (Шаг 1) — применяй его к подписи **полнее, чем к слайдам**, всегда (даже если сами слайды сделаны из вставленного пользователем контента, а не с нуля): тип открывающего крючка, характерный/обязательный закрывающий элемент (например, вопрос к аудитории), место и оформление хэштегов, паттерн использования эмодзи, узнаваемые речевые приёмы из референса — здесь нет ограничения в 25 слов, формат подписи ближе к обычному посту, для которого референс и написан.

## Правила

- Максимум 10 слайдов — жёсткий лимит платформы.
- Видео-слайд: 3-60 секунд, MP4/H.264+AAC, то же соотношение сторон, что у остальных слайдов карусели. В Claude.ai/Cowork видео-режим недоступен.
- У каждого слайда есть alt-текст (для доступности) отдельно от headline/body, и caption (видимая подпись при пролистывании, Instagram Multiple Captions) отдельно от alt-текста и от общей подписи поста — три разных текстовых слоя, не путать и не дублировать между собой.
- Всегда жди подтверждения брифа перед рендером/генерацией промптов.
- Обложка и CTA-слайд визуально отличаются от value-слайдов.
- Палитра и шрифт одинаковы на всех слайдах карусели.
- Без em dash.
- Если в проекте есть `brand-kit.md` — использовать его hex-коды и типографику. Если файла нет и планируется больше одной карусели для этого бренда — предложить создать его из `templates/brand-kit-template.md`.
- Если найден голосовой референс (`voice.md`/`tone-of-voice.md`/явно названный) — при конфликте с жёсткими лимитами карусели (10 слайдов, ≤8/≤25 слов на слайд, без em dash) лимиты карусели всегда побеждают. Голос не даёт права их нарушать.
